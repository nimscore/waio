## `EnumErrorCopy item path`
---
```rust
pub enum Error {
    InvalidToken,
    IoError(Error),
    OtherError(Box<dyn Error + Sync + Send>),
}
```

## Variants

§
### InvalidToken

When an event source is registered (or re- or un-registered) with the
event loop, this error variant will occur if the token Calloop uses to
keep track of the event source is not valid.

§
### IoError(Error)

This variant wraps astd::io::Error, which might arise from
Calloop’s internal operations.

§
### OtherError(Box<dynError+Sync+Send>)

Any other unexpected error kind (most likely from a user implementation ofEventSource::process_events()) will be wrapped in this.

## `EnumModeCopy item path`
---
```rust
pub enum Mode {
    OneShot,
    Level,
    Edge,
}
```

## Variants

§
### OneShot

Single event generation

This FD will be disabled as soon as it has generated one event.

The user will need to useLoopHandle::update()to re-enable it if
desired.

§
### Level

Level-triggering

This FD will report events on every poll as long as the requested interests
are available.

§
### Edge

Edge-triggering

This FD will report events only when itgainsone of the requested interests.
it must thus be fully processed before it’ll generate events again.

This mode is not supported on certain platforms, and an error will be returned
if it is used.

##### Supported Platforms

As of the time of writing, the platforms that support edge triggered polling are
as follows:

- Linux/Android
- macOS/iOS/tvOS/watchOS
- FreeBSD/OpenBSD/NetBSD/DragonflyBSD
